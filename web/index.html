rota2ics

Convert a UK railway-staff rota PDF into a calendar (.ics) file you can import into Google Calendar, Apple Calendar, or Outlook. Everything runs in your browser — your PDF is never uploaded anywhere.

On iPhone or iPad: open this page in Safari. Other iOS browsers can't hand the downloaded .ics file to the Calendar app.

1. Choose your rota PDF

2. Pick your Link and Wk row

A Link is one rotating cycle of weeks; the row you sit on (your Wk) advances by one each week. Tell us which Link you’re on, which row you’re sitting on this week, and which Sunday that week starts.

  • Link — the rota number printed at the top of your PDF (e.g. “Link 1”).
  • Wk row — the numbered row of that Link covering the week starting on the Sunday you choose. The row will advance automatically each week and wrap back to 1 at the end of the cycle.
  • Sunday of that week — must be a Sunday. If you pick another day we’ll use the previous Sunday.
  • Weeks to generate — how far ahead to project events. 26 ≈ 6 months, 52 = a year.

3. Options

  • Calendar name — the name shown when the .ics is imported (Google/Apple/Outlook display this as the calendar or event source).
  • Title prefix — prepended to every event title, useful if you import shifts into a shared calendar (e.g. “Work: BM 112” instead of just “BM 112”).
Rest days (FD)

FD = Free Day on the rota. When ticked, each rest day appears as a transparent all-day event so the day looks free in your calendar but is clearly labelled. Untick if you only want working days in your calendar.

As Ordered (AO)

AO = As Ordered (a.k.a. “spare”): you’re on duty but the actual diagram is assigned on the day. The PDF still prints nominal sign-on / sign-off times — choose how to render those:

Route Refresher (RB)

RB = Route Refresher: a route familiarisation turn. The PDF prints sign-on / sign-off times — choose how to render those. (Only shown because your PDF contains RB days.)

4. Generate

Downloads a .ics file you can import into Google Calendar, Apple Calendar, or Outlook. Times are written as floating local times, so each shift always displays in the viewer’s local time zone (perfect for UK use across BST/GMT).
